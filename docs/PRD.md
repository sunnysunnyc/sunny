# 팬덤 플랫폼 경쟁사 분석 에이전트

## 문제
> 플랫폼 전략팀이 수동 리서치에 의존하던 경쟁사 분석을, 자연어 한 줄로 즉시 실행 가능한 AI 에이전트로 자동화한다

- **현재 상태**: 경쟁사 1개 분석에 3~4시간 소요, 분석 프레임 비일관, 월 1회 업데이트가 한계, 경영진 보고용 재가공에 추가 2시간
- **원하는 상태**: "위버스 최근 어때?" 한 마디로 최신 분석이 나오고, 5개사 비교 대시보드가 자동 갱신되며, 전략 의사결정에 바로 쓸 수 있는 리포트가 3분 내 생성
- **성공 기준**:
  - 경쟁사 분석 소요시간 **3~4시간 → 5분 이내** (95% 단축)
  - 분석 커버리지 **5개사 x 7개 분석 항목** 구조화 데이터 상시 유지

## 스킬
| # | 스킬명 | 한 줄 설명 | 상태 |
|---|--------|-----------|------|
| 1 | `/analyze-competitor` | 경쟁사명 입력 → 웹 검색 → JSON 데이터 자동 업데이트 → 인사이트 보고 | ✅ 동작 |
| 2 | `/compare-competitors` | 2개 이상 경쟁사 다차원 비교 → 포지셔닝 맵 + 기능 매트릭스 리포트 생성 | ✅ 동작 |
| 3 | `/generate-report` | 비교분석/주간브리핑/경영진요약 3종 템플릿 기반 리포트 자동 생성 | ✅ 동작 |
| 4 | `/market-update` | 팬덤 플랫폼 시장 최신 동향 수집 → 카테고리별 정리 → 브리핑 | ✅ 동작 |
| 5 | `/dashboard` | 5개사 데이터 기반 인터랙티브 HTML 대시보드 자동 생성 | ✅ 동작 |

## 변화 기록
- **Day 1**: "경쟁사 분석을 자동화하는 에이전트를 만들고 싶다" → CLAUDE.md 설정, 프로젝트 구조 설계, analyze-competitor 스킬 생성, 위버스·버블 데이터 첫 수집
- **Day 3**: "5개사 분석 체계 완성 + 자연어로 쓸 수 있는 에이전트" → 비마이프렌즈 추가 발견, 커머스/티켓팅/스포츠IP까지 분석 범위 확장, 대시보드 자동화 완성
- **가장 크게 달라진 점**: 처음엔 "4개사 B2C 앱 비교"였으나, 분석 과정에서 B2B SaaS 모델(비마이프렌즈)이 핵심 경쟁자임을 발견 → 시장 구조 자체의 이해가 바뀜. **에이전트가 단순 정보 수집이 아니라 전략적 시야를 넓혀주는 도구**가 됨.
