# 경쟁사 비교 분석 스킬

## 입력
- `$ARGUMENTS`: 비교할 경쟁사 이름들 (공백 구분) 또는 "all"
  - 예: `weverse bubble` 또는 `all`

## 경쟁사명 매핑
- 위버스, Weverse → weverse
- 버블, Bubble, DearU → bubble
- 베리즈, Berries, Berriz → berries
- 포닝, Phoning → phoning

## 비교 프로세스

### Step 1: 데이터 로드
1. `$ARGUMENTS`가 "all"이면 `data/competitors/` 디렉토리의 모든 JSON 파일을 읽는다
2. 특정 경쟁사가 지정되면 해당 파일들만 읽는다
3. `_schema.md` 파일은 제외한다
4. 데이터가 없는 경쟁사가 있으면 사용자에게 `/analyze-competitor [이름]`을 먼저 실행하라고 안내한다
5. 각 파일의 `meta.last_updated`를 확인하여 7일 이상 오래된 데이터는 경고한다

### Step 2: 다차원 비교 분석

#### 2-1. 제품/기능 비교 매트릭스
`product.core_features`를 카테고리별로 크로스 비교하여 표로 작성:
- 카테고리: social, messaging, commerce, live-streaming, content, membership, ai
- 각 경쟁사의 보유 여부, 유/무료, 차별화 포인트 표시

#### 2-2. 사업 모델 비교
`business.revenue_model`을 기반으로:
- 수익원 다양성 비교
- 구독 vs 커머스 vs 광고 구조 비교
- 가격 정책 비교

#### 2-3. 시장 지표 비교
`market` 데이터 기반:
- MAU 랭킹 (공개된 경우)
- 글로벌 확장 현황
- 앱스토어 평점

#### 2-4. SWOT 크로스 분석
각 경쟁사의 `competitive_position`을 종합하여:
- 누가 어떤 영역에서 강한지
- 공통 위협은 무엇인지
- 기회 영역의 교차점

### Step 3: 인사이트 도출
단순 비교가 아닌, 전략적 시사점을 반드시 도출한다:

1. **시장 포지셔닝 맵**: 각 경쟁사의 포지션을 2x2 매트릭스로 표현
   - X축: 기능 범위 (단일기능 ↔ 슈퍼앱)
   - Y축: 과금 강도 (무료 중심 ↔ 유료 중심)

2. **경쟁 역학**: 누가 누구를 위협하는지, 어디서 충돌이 일어나는지

3. **차별화 기회**: 아직 어떤 플레이어도 점유하지 않은 영역

4. **향후 전망**: 각 플레이어의 다음 움직임 예측

### Step 4: 리포트 저장
1. `templates/comparison-report.md` 템플릿을 읽는다
2. 분석 결과를 템플릿에 채워 넣는다
3. `reports/comparison-[YYYY-MM-DD].md` 파일로 저장한다

### Step 5: 결과 요약
분석의 핵심 내용을 사용자에게 요약하여 보여준다.
리포트 파일 경로를 안내한다.

## 출력 형식

```
## 팬덤 플랫폼 경쟁사 비교 분석

### 📊 기능 비교 매트릭스
| 기능 | Weverse | Bubble | Berries | Phoning |
|------|---------|--------|---------|---------|
| ... | ... | ... | ... | ... |

### 💰 사업 모델 비교
(표 형태)

### 📈 시장 지표 비교
(표 형태)

### 🗺️ 포지셔닝 맵
(2x2 매트릭스 텍스트)

### 💡 전략적 시사점
1. ...
2. ...
3. ...

### ⚔️ 경쟁 역학
(분석)

### 🎯 차별화 기회
(분석)

📁 전체 리포트: reports/comparison-[날짜].md
```

## 주의사항
- 데이터가 "(미확인)"인 항목은 비교 시 명확히 표시한다
- 단순 나열이 아닌, "So What?" (그래서 우리에게 어떤 의미인가)를 반드시 포함한다
- 분석 결과를 바탕으로 구체적인 "권장 액션"을 제시한다
