# 경쟁사 분석 스킬

## 입력
- `$ARGUMENTS`: 분석할 경쟁사 이름

## 경쟁사명 매핑
- 위버스, Weverse → weverse
- 버블, Bubble → bubble
- 유니버스, Universe → universe
- 포닝, Phoning → phoning
- 팹, Fab → fab

## 분석 프로세스

### Step 1: 기존 데이터 확인
1. `data/competitors/$ARGUMENTS.json` 파일이 있으면 읽어서 현재 데이터를 파악한다
2. 없으면 `data/competitors/_schema.md`를 읽고 빈 JSON 구조를 생성할 준비를 한다
3. `last_updated` 날짜를 확인하여 얼마나 오래된 데이터인지 파악한다

### Step 2: 웹 검색으로 최신 정보 수집
아래 주제별로 웹 검색을 실행한다 (한국어 + 영어 모두):

1. **제품/서비스 업데이트**: "[경쟁사명] 앱 업데이트 2026", "[경쟁사명] new features 2026"
2. **사업/재무 현황**: "[경쟁사명] 매출 실적", "[모회사명] revenue earnings"
3. **사용자 동향**: "[경쟁사명] MAU 사용자 수", "[경쟁사명] 앱 리뷰 반응"
4. **전략적 움직임**: "[경쟁사명] 파트너십 투자", "[경쟁사명] expansion strategy"
5. **경쟁 구도**: "[경쟁사명] vs [주요 경쟁사]"

### Step 3: 데이터 구조화 및 저장
1. 수집된 정보를 `data/competitors/_schema.md`의 JSON 스키마에 맞게 정리한다
2. 기존 데이터가 있으면 변경/추가된 부분만 업데이트한다
3. `meta.last_updated`를 오늘 날짜로 갱신한다
4. `meta.sources`에 참고한 출처를 모두 추가한다
5. `trends`에 새로운 동향을 추가한다 (기존 항목은 유지, append-only)
6. `meta.data_confidence`를 업데이트한다
7. `data/competitors/$ARGUMENTS.json` 파일로 저장한다

### Step 4: 결과 보고
아래 형식으로 분석 결과를 사용자에게 보고한다:

```
## [경쟁사명] 분석 완료

### 📌 핵심 요약
- [한 줄 요약]

### 🔍 주요 발견사항
1. [발견 1]
2. [발견 2]
3. [발견 3]

### 📊 주요 데이터
| 항목 | 값 | 출처 |
|------|-----|------|
| MAU | ... | ... |
| 주요 수익모델 | ... | ... |
| 앱스토어 평점 | ... | ... |

### 💡 전략적 시사점
- [시사점 1]
- [시사점 2]

### ⚠️ 데이터 신뢰도
- 확인된 정보: N건
- 추정/미확인 정보: N건
```

## 주의사항
- 확인되지 않은 정보는 반드시 ⚠️ 표시와 함께 "(미확인)" 태그를 붙인다
- 정량 데이터(MAU, 매출 등)는 반드시 출처와 기준 시점을 명시한다
- 검색 결과가 부족하면 사용자에게 알리고, 추가 검색 키워드를 제안한다
- 한국어 우선으로 보고하되, 고유명사와 기술 용어는 원어를 유지한다
